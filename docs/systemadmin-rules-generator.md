# System Admin Rules Generator

## Overview

This document describes how a System Admin creates, edits, and manages award rule combinations for tenants. Rules are derived by joining multiple entities from the Fair Work Awards data model to create comprehensive pay rules applicable to specific industry types.

## SQL Join Examples for Rule Generation

### 1. Get All Classifications with Base Pay Rates for an Award

```sql
-- Join: Classification â†’ PayRate
-- Purpose: List all employee levels with their hourly/weekly rates

SELECT 
    c.classification_fixed_id,
    c.classification AS classification_name,
    c.classification_level,
    c.employee_rate_type_code,
    p.base_pay_rate_id,
    p.base_rate_type,
    p.base_rate,
    p.calculated_rate AS hourly_rate
FROM Classifications c
INNER JOIN PayRates p 
    ON c.classification_fixed_id = p.classification_fixed_id
    AND c.employee_rate_type_code = p.employee_rate_type_code
WHERE c.operative_from <= CURRENT_DATE
    AND (c.operative_to IS NULL OR c.operative_to > CURRENT_DATE)
ORDER BY c.classification_level, c.employee_rate_type_code;
```

**Result Example:**

| classification_name | level | emp_type | weekly_rate | hourly_rate |
|---------------------|-------|----------|-------------|-------------|
| Retail Employee Level 1 | 1 | AD | $1,008.90 | $26.55 |
| 16 years of age | 1 | JN | $504.45 | $13.28 |
| 1st year | 1 | AA | $854.72 | $22.49 |
| 1st year | 1 | AP | $534.20 | $14.06 |

### 2. Calculate Complete Pay with Penalty Rates

```sql
-- Join: Classification â†’ PayRate â†’ PenaltyRate
-- Purpose: Calculate actual pay for specific work conditions

SELECT 
    c.classification AS classification_name,
    c.classification_level,
    pr.base_rate AS base_weekly,
    pen.penalty_description,
    pen.rate AS penalty_percent,
    pen.penalty_calculated_value AS calculated_pay,
    pen.clause_description
FROM Classifications c
INNER JOIN PayRates pr 
    ON c.classification_fixed_id = pr.classification_fixed_id
INNER JOIN PenaltyRates pen 
    ON pr.base_pay_rate_id = pen.base_pay_rate_id
    AND c.classification_level = pen.classification_level
WHERE c.employee_rate_type_code = 'AD'
    AND pen.penalty_description LIKE '%Saturday%'
    AND c.operative_from = '2025-07-01'
ORDER BY c.classification_level, pen.rate;
```

**Result Example (Saturday Penalty Rates):**

| classification | level | base_weekly | penalty_desc | rate% | calculated |
|----------------|-------|-------------|--------------|-------|------------|
| Retail Level 1 | 1 | $1,008.90 | Saturday First 4hrs | 150% | $39.83 |
| Retail Level 1 | 1 | Saturday After 4hrs | 200% | $53.10 |
| Retail Level 1 | 1 | Sunday | 200% | $53.10 |
| Retail Level 1 | 1 | Public Holiday | 300% | $79.65 |

### 3. Calculate Wage Allowances Based on Classification

```sql
-- Join: Classification â†’ PayRate â†’ WageAllowance
-- Purpose: Calculate location/role-based allowances

SELECT 
    c.classification AS classification_name,
    pr.calculated_rate AS hourly_rate,
    wa.allowance AS allowance_name,
    wa.rate AS percentage_rate,
    wa.rate_unit,
    wa.allowance_amount,
    wa.payment_frequency,
    wa.clauses AS award_clause
FROM Classifications c
INNER JOIN PayRates pr 
    ON c.classification_fixed_id = pr.classification_fixed_id
INNER JOIN WageAllowances wa 
    ON pr.base_pay_rate_id = wa.base_pay_rate_id
WHERE c.employee_rate_type_code = 'AD'
    AND c.classification_level = 4  -- Level 4 reference
    AND wa.operative_from = '2025-07-01'
ORDER BY wa.allowance;
```

### 4. Complete Rule Generation Query for a Tenant

```sql
-- Multi-table Join: Industry â†’ Award â†’ Classification â†’ PayRate â†’ Penalty â†’ Allowances
-- Purpose: Generate complete ruleset for a specific industry tenant

SELECT 
    -- Industry & Award
    i.name AS industry_name,
    a.code AS award_code,
    a.name AS award_name,
    
    -- Classification
    c.classification AS classification_name,
    c.classification_level,
    c.employee_rate_type_code,
    
    -- Base Pay
    pr.base_rate_type,
    pr.base_rate,
    pr.calculated_rate AS hourly_rate,
    
    -- Penalties (aggregated)
    pen.penalty_description,
    pen.rate AS penalty_rate,
    pen.penalty_calculated_value,
    
    -- Allowances (expense)
    ea.allowance AS expense_allowance,
    ea.allowance_amount,
    ea.payment_frequency,
    
    -- Allowances (wage-based)
    wa.allowance AS wage_allowance,
    wa.rate AS wage_allowance_rate,
    wa.allowance_amount AS wage_allowance_amount

FROM Industries i
INNER JOIN IndustryAwardMapping iam ON i.industry_id = iam.industry_id
INNER JOIN Awards a ON iam.award_id = a.award_id
INNER JOIN Classifications c ON a.award_id = c.award_id
INNER JOIN PayRates pr ON c.classification_fixed_id = pr.classification_fixed_id
LEFT JOIN PenaltyRates pen ON pr.base_pay_rate_id = pen.base_pay_rate_id
LEFT JOIN ExpenseAllowances ea ON a.award_id = ea.award_id
LEFT JOIN WageAllowances wa ON pr.base_pay_rate_id = wa.base_pay_rate_id

WHERE i.name = 'Retail'
    AND c.operative_from = '2025-07-01'
    AND c.employee_rate_type_code = 'AD';
```

## Summary

The System Admin Rules Generator allows creating comprehensive pay rules by:

1. **Selecting an Award** based on industry type
2. **Configuring Classifications** (levels and employee types)
3. **Enabling Penalty Rates** for various work conditions
4. **Activating Allowances** (expense and wage-based)
5. **Joining these entities** to generate a complete ruleset
6. **Assigning rulesets** to tenants with effective dates
7. **Editing and versioning** rules as FWC updates occur

All rules are derived from authoritative FWC data and maintain compliance with Fair Work Award requirements.
