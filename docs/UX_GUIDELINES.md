# UX Design Guidelines

## Fair Work Awards Management System

This document provides UX guidelines and wireframe suggestions for designing the Fair Work Awards Management System, focusing on System Admin and Tenant Admin interfaces.

---

## Design Principles

### 1. Clarity First
Award data is complex and legally binding. Every interface should prioritize:
- Clear labeling and terminology
- Obvious data hierarchy
- Unambiguous actions

### 2. Progressive Disclosure
Don't overwhelm users with all data at once:
- Show summary first, details on demand
- Use expandable sections
- Implement drilling down from overview to detail

### 3. Compliance-Driven Design
Users need to trust the data:
- Always show effective dates
- Display data source (FWC)
- Provide audit trails
- Make version history accessible

### 4. Mobile-Responsive
Tenant Admins may need to check rates on-the-go:
- Design mobile-first for rate lookups
- Keep complex configuration desktop-optimized

## Component Specifications

### Data Tables

**Rate Table:**
- Sticky header for scrolling
- Sortable columns
- Row hover highlight
- Export options (PDF, Excel, CSV)
- Responsive: horizontal scroll on mobile

**Color Coding:**
- Current rates: Default text
- Upcoming changes: Blue highlight with icon
- Expired rates: Grey/muted
- Above-award rates: Green indicator

### Cards & Metrics

**Metric Cards:**
- Large number (32px+)
- Label below (14px)
- Trend indicator when relevant
- Click to drill down

### Forms

**Input Fields:**
- Clear labels above fields
- Helpful placeholder text
- Inline validation
- Error messages below field

**Selection:**
- Dropdowns for <10 options
- Searchable select for >10 options
- Radio buttons for 2-4 mutually exclusive options
- Checkboxes for multi-select

### Notifications

**Alert Banner:**
- Top of page, dismissible
- Color coded: Info (blue), Warning (yellow), Error (red), Success (green)

**Toast Notifications:**
- Bottom right corner
- Auto-dismiss after 5 seconds
- Action link when relevant

## Accessibility Guidelines

### WCAG 2.1 AA Compliance

1. **Color Contrast**: Minimum 4.5:1 for normal text
2. **Focus States**: Visible focus indicators on all interactive elements
3. **Screen Readers**: All images have alt text, tables have headers
4. **Keyboard Navigation**: All functions accessible via keyboard
5. **Form Labels**: All inputs have associated labels

### Responsive Breakpoints

| Breakpoint | Width | Layout |
|------------|-------|--------|
| Mobile | <768px | Single column, stacked |
| Tablet | 768-1024px | Two columns, sidebar collapsed |
| Desktop | >1024px | Full layout with sidebar |
